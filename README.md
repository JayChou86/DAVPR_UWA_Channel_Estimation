# Time-Varying Underwater Acoustic Channel Estimation & DAVPR Receiver Design

Deep-sea time-varying channel estimation and Depth-Adaptive Variable-Parameter Receiver (DAVPR) design for AUV communications in main thermoclines.

## Project Structure

```
├── window_*fenduan.m         # Sliding-window CIR estimation from raw WAV signals
├── Channel_review_*.m        # Channel replay & BER simulation with RLS-DFE
├── TimeVeryCharater.m        # Channel statistical characterization (coherence time/bandwidth, Doppler, PDP)
├── plot_*.m                  # Visualization utilities (timecor, freqcor, doppler, rician)
├── all_PDF.m / PDFfangzhen.m # Statistical distribution analysis of CIR amplitudes
├── meanCIR.m                 # Mean CIR 3D comparison across depths
├── pre_process.m             # Quick CIR preview utility
├── .gitignore                # Excludes large regeneratable .mat files
└── Depth-Adaptive_.../       # Related paper PDF and figures
```

## Pipeline

### 1. Channel Estimation (`window_*fenduan.m`)
- Reads raw WAV signals: transmitted probe, received signal, calibration signal
- Downsampling to GCD sample rate → bandpass filter (600–1000 Hz)
- Cross-correlation synchronization
- Sliding-window segmentation → FFT-based frequency-domain cross-correlation
- Output: time-varying CIR matrix `h_matrix` (74 time slices × 95999 delay taps, Δt = 0.1 s)

### 2. Channel Characterization (`TimeVeryCharater.m`)
- Time autocorrelation → coherence time
- Frequency autocorrelation → coherence bandwidth
- Doppler power spectrum → Doppler spread (centroid + RMS)
- Delay-Doppler power spectrum
- Power delay profile → mean delay + RMS delay spread

### 3. Communication Performance (`Channel_review_*.m`)
- QPSK modulation (4000 sps, fs = 16 kHz)
- Time-varying convolution (block-wise overlap-add)
- RLS Decision Feedback Equalizer (`comm.DecisionFeedbackEqualizer`)
- BER vs. SNR evaluation (0–30 dB)

## Reproducing Results

1. Run `window_475fenduan.m` (and 624/824/924 variants) to estimate CIR from raw WAV recordings
2. Run `TimeVeryCharater.m` for channel statistics (modify `load()` call for each depth)
3. Run `Channel_review_475.m` (and variants) for BER performance evaluation
